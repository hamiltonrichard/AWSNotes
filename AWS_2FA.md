# AWS 2FA for Windows and Linux

Securing both your **AWS Root User** and your **Administrator account** (ideally managed via **AWS IAM Identity Center**) with Multi-Factor Authentication (MFA) is a critical step in hardening your cloud infrastructure.

Here is how to set up 2FA for both account types, featuring robust, local token-management options for both Windows and openSUSE Linux platforms so you don't have to rely solely on a smartphone app.

---

## Part 1: Enrolling MFA in the AWS Console

Before setting up your local client, you need to initiate the device assignment within AWS.

### 1. The Account Root User

1. Sign in to the AWS Management Console using your root email address.
2. In the top-right corner, click your account name and select **Security credentials**.
3. Scroll down to the **Multi-factor authentication (MFA)** section and click **Assign MFA device**.
4. Select **Authenticator app** (or *Passkey/Security Key* if you are using a physical FIDO2 device like a Yubikey) and click Next.
5. Click **Show QR code** or click **Show secret key** to reveal the alphanumeric base32 string. **Keep this open.**

### 2. The IAM Identity Center Administrator Account

> *Best Practice Note:* For daily administrative work, you should use AWS IAM Identity Center (formerly AWS SSO) rather than long-term IAM users.

1. Sign into your **AWS Access Portal** as an Administrator.
2. Go to your portal profile settings or, if configuring it globally as the Org administrator, navigate to the **IAM Identity Center Console** -> **Settings** -> **MFA**.
3. Under user settings, trigger **Register MFA device** and choose **Authenticator app** to expose the secret seed configuration.

## Part 2: Platform-Specific Client Options

AWS uses standard Time-Based One-Time Password (TOTP) algorithms ($RFC\ 6238$). You can use cross-platform password managers like **Bitwarden** or **1Password** on either OS to automatically sync and generate these tokens, but if you prefer dedicated local desktop utilities, use the implementations below.

### Option A: OpenSUSE Linux

OpenSUSE offers powerful, secure terminal and GUI utilities to manage TOTP keys entirely locally.

#### 1. The GUI Method: KeePassXC

**KeePassXC** is an exceptional, offline credential vault that natively calculates TOTP tokens.

* **Installation:** ```bash
sudo zypper in keepassxc
```

```


* **Configuration:** Open your database, create a new entry for AWS, right-click the entry, select **TOTP** -> **Set up TOTP...**, and paste the alphanumeric **Secret Key** provided by AWS. It will display a rolling 6-digit code every 30 seconds.

#### 2. The CLI Method: `oathtool`

If you manage your environments entirely from the shell or want to script your workflow tokens, `oathtool` is a lightweight utility from the GNU OATH project.

* **Installation:**
```bash
sudo zypper in oathtool

```


* **Usage:** Securely save your plaintext secret string. You can instantly generate a current token by running:
```bash
oathtool --totp -b "YOUR_AWS_SECRET_KEY_HERE"

```

### Option B: Windows

Windows provides native applications as well as lightweight open-source utilities to handle local verification codes.

#### 1. The Native GUI Method: KeePassXC (Windows Port)

Just like on Linux, the Windows native port of **KeePassXC** allows you to keep an encrypted, offline database (`.kdbx`) locally.

* **Setup:** Download the installer from the official site or install via winget (`winget install KeePassXCTeam.KeePassXC`). Follow the same right-click entry -> **Set up TOTP** workflow to inject your AWS secret key.

#### 2. The Windows Store App: Authenticator Authenticator

If you want a dedicated UI similar to a mobile application layout on your desktop, **Authenticator** (by Quentin Chateau, available on GitHub/Windows Store) is an excellent choice.

* **Setup:** Click the "+" button, type "AWS Root" or "AWS Admin", and paste the secret token or use your webcam to scan the desktop QR code.

## Part 3: Finalizing the Link with AWS

Once your client (on either Linux or Windows) begins spitting out 6-digit rolling codes:

1. Return to the AWS browser tab.
2. Input **two consecutive codes** generated by your tool into the **MFA code 1** and **MFA code 2** fields. AWS requires this sequence to verify that your system clock is properly synchronized with their authentication servers.
3. Click **Assign MFA**.

> ⚠️ **Critical Backup Strategy:** AWS allows you to register up to **8 MFA devices** per user. It is highly recommended to bind a secondary method (such as a physical FIDO2 USB Security Key or an authenticator on a completely separate device) so you are never locked out of the root environment if a local database corruption occurs.
